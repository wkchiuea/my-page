Object-oriented Programming（OOP，物件導向程式設計）是一種以物件為中心的程式設計方法。OOP 透過**封裝（Encapsulation）**、**抽象（Abstraction）**、**繼承（Inheritance）**與**多型（Polymorphism）**這四大支柱，讓程式架構更加模組化、易於維護與擴展。在 OOP 中，物件（Object）是從類別（Class）實例化而來，代表現實世界中的事物或概念。

## OOP 的四大支柱

1. **封裝（Encapsulation）**  
   將資料（屬性）和行為（方法）包裝在同一個類別中，外部只能透過類別提供的介面與物件互動，隱藏內部實作細節。

2. **抽象（Abstraction）**  
   只對外暴露必要的介面，隱藏複雜的實作邏輯。透過抽象，我們可以專注於「做什麼」而非「怎麼做」，降低使用者的認知負擔。

3. **繼承（Inheritance）**  
   允許新類別繼承現有類別的屬性及方法，實現程式碼重用，形成階層結構。

4. **多型（Polymorphism）**  
   允許同一方法名稱在不同類別中有不同實現，達到介面的統一與彈性。

## 簡單範例

以下分別使用 Python 與 JavaScript 實作簡單的 OOP 範例：

### Python 範例

```python
# 定義一個 Animal 類別
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print(f"{self.name} 發出聲音")

# 繼承 Animal，定義 Dog 類別
class Dog(Animal):
    def speak(self):
        print(f"{self.name}：汪！")

# 使用類別
dog = Dog("小黑")
dog.speak()  # 輸出：小黑：汪！
```

### JavaScript 範例

```javascript
// 定義 Animal 類別
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} 發出聲音`);
  }
}

// 繼承 Animal，定義 Dog 類別
class Dog extends Animal {
  speak() {
    console.log(`${this.name}：汪！`);
  }
}

const dog = new Dog("小黑");
dog.speak(); // 輸出：小黑：汪！
```

## 小結

OOP 能幫助你寫出更健壯、結構化的程式。當程式越來越大時，OOP 的封裝、抽象、繼承與多型能讓程式更容易維護與擴展。建議初學者可以從簡單的類別與物件實作開始，逐步熟悉 OOP 的設計思維。
