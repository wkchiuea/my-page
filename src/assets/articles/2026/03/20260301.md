Functional Programming（函數式程式設計）是一種把「函數」視為主要組成單位的程式設計範式。它起源於數學上的 λ-計算（Lambda Calculus），注重用純函數 (pure function) 來處理資料、避免副作用 (side effects)、以及資料不可變性 (immutability)。對比於物件導向程式設計（OOP）的以「物件」為中心，函數式編程強調資料的流動和資訊的轉換，常見語言有：Haskell、Lisp、Scala、F#，以及 Python、JavaScript 等多范式語言的支援。

---

## 函數式程式設計的核心概念

1. **純函數 (Pure Function)**  
   純函數的輸出僅決定於輸入參數，不依賴也不改變外部狀態（無副作用）。這有助於程式可預測和易於測試。

2. **不可變性 (Immutability)**  
   資料一經創建即不可變。修改資料時，會回傳新的資料副本，確保狀態不被意外改動。

3. **高階函數 (Higher-Order Functions)**  
   函數可以作為參數傳遞給其他函數，或作為回傳值。這讓程式更具彈性和可重用性。

4. **宣告式程式設計 (Declarative Programming)**  
   強調「做什麼」（what），而不是「怎麼做」（how）；更關注運算結果而非具體的執行過程。

## 簡單範例

以下是一些使用函數式程式設計風格的 Python 和 JavaScript 範例：

### Python 範例

```python
# 一個純函數
def add(x, y):
    return x + y

# 高階函數: map
nums = [1, 2, 3, 4]
squared = list(map(lambda x: x ** 2, nums))  # [1, 4, 9, 16]

# 使用 filter 過濾偶數
evens = list(filter(lambda x: x % 2 == 0, nums))  # [2, 4]
```

---

### JavaScript 範例

```javascript
// 純函數
const add = (x, y) => x + y;

// 高階函數: map
const nums = [1, 2, 3, 4];
const squared = nums.map(x => x ** 2); // [1, 4, 9, 16]

// 使用 filter 過濾偶數
const evens = nums.filter(x => x % 2 === 0); // [2, 4]
```

## 與命令式/物件導向的差異

- **函數式**：強調「資料流變換」，用組合多個小且純的函數處理資料，結果為新資料，不改動原資料。
- **命令式/物件導向**：頻繁改變狀態、物件屬性，操作較靠近計算機執行順序。

---

## 優點與挑戰

### 優點
- 程式更易於測試與除錯（純函數、無副作用）。
- 更容易寫出「並行/平行程式」，數據不共享，安全性高。
- 可重用性高，方便組裝複雜邏輯。

### 挑戰
- 思維轉換需要時間，初學者容易卡在不可變與組合的概念上。
- 在處理大量 I/O、需要強狀態管理時，容易比命令式程式碼複雜。

---

## 小結

函數式程式設計提供了解決複雜軟體問題的另一種強大方法。即使你用的語言不是純粹的函數式語言（像是 JavaScript、Python），仍可以在日常開發中導入純函數、不變性和高階函數，提升程式的可讀性與穩健性。建議可以從嘗試寫純函數開始，也多利用 map/filter/reduce 等內建函數練習。
